# Lessons Learned

## デザイン・表示確認の品質基準

### 絶対にやること
- スクリーンショットを撮ったら、以下の観点で自己チェックする：
  1. テキストが切れていないか、はみ出していないか
  2. 要素同士が重なっていないか
  3. 意図しない線や境界が見えていないか
  4. 余白は適切か（詰まりすぎ・空きすぎ）
  5. 色やフォントが指定通りか
- 問題を発見したら、ユーザーに報告する前に自分で修正を試みる
- 「確認済みです」と言う前に、上記5点を本当にチェックしたか自問する

### 絶対にやらないこと
- スクリーンショットを撮っただけで「問題ない」と判断しない
- 明らかに表示がおかしいのに「動作しています」と報告しない
- ユーザーの指摘に対して「既に確認済み」と安易に返さない
- 吹き出し等の複雑なUI要素で、重なり・はみ出し・切れを見逃さない

### SVG吹き出し実装の教訓
- SVGの三角+四角の接合部は必ず境界線の問題が発生する → 三角を使わずコネクタ線を使う
- RechartsのCustomizedコンポーネントはscale情報が取得できない場合がある → ReferenceDotのshapeを使う
- 複数要素の重なり回避は、placed配列で座標を追跡して衝突判定する
